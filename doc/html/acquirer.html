Acquirer

An acquirer is a lock monitor. It takes a mutex argument in its constructor. This mutex is locked when the acquirer is constructed and unlocked when the acquirer gets destroyed. Thus the mutex is locked while the acquiere is in scope and unlocked when the monitor gets out of scope.

Acquirers are used to avoid complicated mutex lock and unlock call sequences in functions. When a mutex is placed in an acquirer at the begin of a code block it will be locked until the code block exits either normally, with a return, or because of an exception.

~acquirer ()

Destroys this acquirer, releasing the lock on the mutex argument given in the constructor.

acquirer (mutex& monitor)

Constructs a new acquirer, locking the mutex argument.

  • deadlock is thrown if the calling thread already owns the mutex.
