Releaser

A releaser is a unlock monitor. It takes a mutex argument in its constructor. This mutex is unlocked when the releaser is constructed and locked when the releaser gets destroyed. Thus the mutex is unlocked while the releaser is in scope and locked when the monitor gets out of scope.

Releasers are used to avoid complicated mutex unlock and lock call sequences in functions. When a mutex is placed in a releaser at the begin of a code block it will be unlocked until the code block exits either normally, with a return, or because of an exception.

~releaser ()

Destroys this releaser, acquiring the lock on the mutex argument given in the constructor.

releaser (mutex& monitor)

Constructs a new releaser, unlocking the mutex argument.

  • permission is thrown if the calling thread doesn't own the mutex.
